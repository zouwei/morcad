> **文档版本**：v1.0
> **日期**：2026-03-11
> **状态**：规划中
> **关联项目**：[Moraya](https://github.com/your-org/moraya)

## 1. 项目概述

### 1.1 项目名称

**`morcad`**（GitHub 仓库名）

npm 包名规划：

| 包名 | 说明 |
|------|------|
| `@morcad/core` | 共享渲染核心（GPLv3） |
| `@morcad/moraya` | Moraya 适配层（GPLv3） |
| `@morcad/obsidian` | Obsidian 适配层（GPLv3） |

### 1.2 项目定位

一个独立分发的 CAD 文件代码块渲染插件，支持在 Markdown 编辑器中将 `.dxf`（直接支持）/ `.dwg`（通过 WASM 转换）工程图纸以代码块形式内嵌渲染，无需打开独立窗口。

**核心价值主张：**

> "在 Markdown 文档中直接嵌入 CAD 工程图，就像嵌入 Mermaid 流程图一样自然。"

### 1.3 目标宿主平台

| 平台 | 集成方式 | 优先级 |
|------|---------|--------|
| **Moraya** | `RendererPlugin` 条目（`renderer-registry.ts`） | P0 主要目标 |
| **Obsidian** | `registerMarkdownCodeBlockProcessor` | P1 兼容目标 |
| 其他编辑器 | 通过共享核心包适配 | P2 未来扩展 |

## 2. 许可证策略

### 2.1 为什么整体采用 GPLv3

本插件的 DWG 解析能力依赖 `@mlightcad/libredwg-web`，该包基于 LibreDWG（GNU GPLv3）编译为 WebAssembly。根据 FSF 许可证规则：

- GPLv3 代码不能并入 Apache-2.0 项目，**但可以独立分发**
- **分发隔离原则**：Moraya 主包（Apache-2.0）不包含本插件代码，用户主动下载本插件，本插件独立遵守 GPLv3

**Apache-2.0 与 GPLv3 的单向兼容规则：**

```
✅  Apache-2.0 代码  →  可以并入 GPLv3 项目（合并体变为 GPLv3）
❌  GPLv3 代码      →  不能并入 Apache-2.0 项目
```

### 2.2 分发隔离法律结构

```
Moraya 主应用（Apache-2.0）
    │ 运行时动态加载（用户主动触发下载）
    ▼
@morcad/moraya（GPLv3）            ← 本仓库，独立分发
    ├── @morcad/core（GPLv3）
    │     ├── dxf-parser（MIT）     ← MIT 兼容 GPLv3 ✓
    │     ├── three-dxf-loader（MIT）← MIT 兼容 GPLv3 ✓
    │     ├── three（MIT）           ← MIT 兼容 GPLv3 ✓
    │     └── @mlightcad/libredwg-web（GPLv3）← 驱动整体 GPLv3
```

Moraya 插件页面注明："DWG 支持由独立 GPLv3 插件（MorCad）提供，与 Moraya 主程序许可证独立。"

### 2.3 LICENSE 文件

仓库根目录放置标准 `GNU General Public License v3.0` 全文。

## 3. 用户交互设计

### 3.1 代码块语法

**基础用法（文件路径引用）：**

````markdown
```dxf
./drawings/floor-plan.dxf
```
````

**DWG 格式（相同语法，自动识别扩展名）：**

````markdown
```dxf
./drawings/building.dwg
```
````

**带参数用法（v0.2.0 实现）：**

````markdown
```dxf
file: ./electrical-plan.dxf
layers: WALLS, DOORS, ELECTRICAL
height: 500
theme: auto
```
````

**语法规则：**

- 代码块内容为**相对路径**，相对于当前 Markdown 文档所在目录
- 支持 `.dxf`（直接解析，MIT 栈，无 WASM）和 `.dwg`（WASM 转换后解析，GPLv3 组件懒加载）
- 参数解析：首行若为 `key: value` 格式则按参数解析，否则整个内容视为文件路径

### 3.2 渲染效果规格

| 特性 | 说明 |
|------|------|
| 渲染技术 | Three.js WebGL，正交相机（2D 工程图专用） |
| 默认高度 | 400px，可通过 `height` 参数调整 |
| 交互 | 鼠标/触控缩放（滚轮/双指）、平移（拖拽），**禁用 3D 旋转** |
| 图层面板 | 可折叠浮层，列出所有图层，支持单独显示/隐藏 |
| 主题 | 跟随宿主编辑器深色/浅色模式自动切换 |
| 加载态 | 骨架屏 + 进度文字（DWG 需额外提示 WASM 加载） |
| 错误态 | 友好错误信息 + 文件路径显示 |

## 4. 技术架构

### 4.1 技术选型

| 层次 | 选型 | 许可证 | 说明 |
|------|------|--------|------|
| DXF 解析 | `dxf-parser` v1.1.2 | MIT | 纯 JS，无 WASM，稳定 |
| DWG 解析 | `@mlightcad/libredwg-web` v0.6.6 | GPLv3 | LibreDWG 编译为 WASM |
| Three.js 集成 | `three-dxf-loader` v5.2.0 | MIT | 活跃维护，DXF→Three.js Mesh |
| 渲染引擎 | `three` v0.170.0 | MIT | WebGL，正交相机 |
| 相机控制 | `OrbitControls`（Three.js addons） | MIT | 限制为 2D 模式 |
| 构建工具 | `vite` + `typescript` | MIT | ESM 输出 |
| 包管理 | `pnpm` workspace | — | Monorepo |

### 4.2 为什么用正交相机

CAD 工程图为严格 2D 内容，要求平行线保持平行（无透视缩短）。`THREE.OrthographicCamera` 是唯一正确选择，`PerspectiveCamera` 会产生透视变形导致图纸失真。

## 5. 核心类型接口

```typescript
// packages/core/src/types.ts

/** 解析后的统一 CAD 文档数据结构 */
export interface CadDocument {
  layers: CadLayer[];
  entities: CadEntity[];
  boundingBox: BoundingBox;
  units: 'metric' | 'imperial' | 'unknown';
  sourceFormat: 'dxf' | 'dwg';
}

export interface CadLayer {
  name: string;
  color: number;       // AutoCAD 颜色索引（0-256）
  colorHex: string;    // 已转换的 CSS hex 色值
  visible: boolean;
  frozen: boolean;
  lineWeight: number;
}

export interface BoundingBox {
  minX: number;
  minY: number;
  maxX: number;
  maxY: number;
  width: number;
  height: number;
  centerX: number;
  centerY: number;
}

/** 渲染配置 */
export interface CadRenderOptions {
  height?: number;                     // 容器高度（px），默认 400
  theme?: 'light' | 'dark' | 'auto';  // 默认 'auto'
  initialLayers?: string[];           // 初始可见图层（默认全部可见）
  showLayerPanel?: boolean;           // 是否显示图层面板，默认 true
  backgroundColor?: string;           // 覆盖背景色
}

/** 代码块内容解析结果 */
export interface ParsedSource {
  filePath: string;                    // 文件路径（相对）
  options: CadRenderOptions;           // 从参数行解析的配置
}

/** 渲染实例（外部控制句柄） */
export interface CadViewerInstance {
  dispose(): void;                             // 销毁，释放 GPU 资源
  setLayerVisibility(layer: string, visible: boolean): void;
  resetCamera(): void;                         // 自动 fit 到 boundingBox
  getLayers(): CadLayer[];
}

/**
 * 平台适配器接口
 * 唯一需要各平台自行实现的部分，隔离文件系统差异
 */
export interface FileReaderAdapter {
  /**
   * 读取文件二进制内容
   * Moraya: invoke('read_file_binary', { path })
   * Obsidian: app.vault.adapter.readBinary(path)
   */
  readFile(absolutePath: string): Promise<Uint8Array>;

  /**
   * 将代码块中的相对路径解析为绝对路径
   * @param relativePath  如 "./drawings/plan.dxf"
   * @param docPath       当前文档绝对路径（null 表示未保存文档）
   */
  resolvePath(relativePath: string, docPath: string | null): string;
}
```

## 7. 渲染管线实现要点

### 7.1 代码块内容解析

```typescript
// packages/core/src/source-parser.ts

export function parseSource(source: string): ParsedSource {
  const lines = source.trim().split('\n');

  // 判断是否为参数模式（任意行包含 ": "）
  const isParameterMode = lines.some(l => /^\w+:\s/.test(l));

  if (!isParameterMode) {
    // 简单模式：整个内容为文件路径
    return { filePath: source.trim(), options: {} };
  }

  // 参数模式：解析 key: value 行
  const params: Record<string, string> = {};
  for (const line of lines) {
    const match = line.match(/^(\w+):\s*(.+)/);
    if (match) params[match[1]] = match[2].trim();
  }

  return {
    filePath: params.file ?? '',
    options: {
      height: params.height ? parseInt(params.height) : undefined,
      theme: params.theme as CadRenderOptions['theme'],
      initialLayers: params.layers?.split(',').map(s => s.trim()),
    }
  };
}
```

### 7.2 DXF 解析

```typescript
// packages/core/src/parser-dxf.ts
import DxfParser from 'dxf-parser';

export function parseDxf(content: string): CadDocument {
  const parser = new DxfParser();
  const raw = parser.parseSync(content);

  // 转换 AutoCAD 颜色索引 → CSS hex（使用 ACI 颜色表）
  const layers = Object.values(raw.tables?.layer?.layers ?? {}).map(layer => ({
    name: layer.name,
    color: layer.color,
    colorHex: aciToHex(layer.color),
    visible: !layer.frozen,
    frozen: layer.frozen ?? false,
    lineWeight: layer.lineWeight ?? 0,
  }));

  return {
    layers,
    entities: raw.entities ?? [],
    boundingBox: computeBoundingBox(raw.entities ?? []),
    units: detectUnits(raw.header),
    sourceFormat: 'dxf',
  };
}
```

### 7.3 DWG 懒加载

```typescript
// packages/core/src/parser-dwg.ts

let libredwgInstance: unknown = null;

export async function parseDwg(content: Uint8Array): Promise<CadDocument> {
  if (!libredwgInstance) {
    // 动态导入：仅遇到 .dwg 文件时才下载 ~5MB WASM
    const mod = await import('@mlightcad/libredwg-web');
    libredwgInstance = await mod.default();
  }

  // libredwg-web: DWG 二进制 → DXF 文本字符串
  const dxfContent = (libredwgInstance as {
    dwgToSvg: (data: Uint8Array) => string;
  }).dwgToSvg(content);

  return parseDxf(dxfContent);
}
```

### 7.4 Three.js 渲染器

```typescript
// packages/core/src/renderer.ts
import * as THREE from 'three';
import { OrbitControls } from 'three/addons/controls/OrbitControls.js';

export class CadRenderer {
  private scene: THREE.Scene;
  private camera: THREE.OrthographicCamera;
  private renderer: THREE.WebGLRenderer;
  private controls: OrbitControls;
  private layerGroups: Map<string, THREE.Group> = new Map();
  private animationId: number | null = null;

  constructor(container: HTMLElement, options: CadRenderOptions) {
    // 场景初始化
    this.scene = new THREE.Scene();
    this.scene.background = new THREE.Color(this.resolveBackground(options));

    // 正交相机（2D 工程图专用）
    const aspect = container.clientWidth / (options.height ?? 400);
    this.camera = new THREE.OrthographicCamera(
      -aspect * 500, aspect * 500, 500, -500, 0.1, 10000
    );
    this.camera.position.set(0, 0, 100);

    // WebGL 渲染器
    this.renderer = new THREE.WebGLRenderer({ antialias: true });
    this.renderer.setPixelRatio(window.devicePixelRatio);
    this.renderer.setSize(container.clientWidth, options.height ?? 400);
    container.appendChild(this.renderer.domElement);

    // OrbitControls：限制为 2D 模式
    this.controls = new OrbitControls(this.camera, this.renderer.domElement);
    this.controls.enableRotate = false;     // 禁用 3D 旋转
    this.controls.enableDamping = true;
    this.controls.dampingFactor = 0.1;
    this.controls.screenSpacePanning = true;

    this.startRenderLoop();
  }

  private startRenderLoop() {
    const animate = () => {
      this.animationId = requestAnimationFrame(animate);
      this.controls.update();
      this.renderer.render(this.scene, this.camera);
    };
    animate();
  }

  async loadDocument(doc: CadDocument): Promise<void> {
    // 为每个图层创建 Group
    for (const layer of doc.layers) {
      const group = new THREE.Group();
      group.name = layer.name;
      group.visible = layer.visible;
      this.layerGroups.set(layer.name, group);
      this.scene.add(group);
    }

    // 使用 three-dxf-loader 将实体转为 Three.js 对象
    // 按 layer 属性分配到对应 Group
    // ...实体渲染逻辑...

    this.fitCameraToDocument(doc.boundingBox);
  }

  private fitCameraToDocument(bb: BoundingBox) {
    // 自动调整相机以完整显示图纸
    const padding = 1.1;
    const halfW = (bb.width * padding) / 2;
    const halfH = (bb.height * padding) / 2;
    const aspect = this.renderer.domElement.width / this.renderer.domElement.height;

    const halfSize = Math.max(halfW / aspect, halfH);
    this.camera.left = -halfSize * aspect;
    this.camera.right = halfSize * aspect;
    this.camera.top = halfSize;
    this.camera.bottom = -halfSize;
    this.camera.position.set(bb.centerX, bb.centerY, 100);
    this.camera.updateProjectionMatrix();
  }

  setLayerVisibility(layerName: string, visible: boolean): void {
    const group = this.layerGroups.get(layerName);
    if (group) group.visible = visible;
  }

  resetCamera(): void {
    this.controls.reset();
  }

  dispose(): void {
    // 取消渲染循环
    if (this.animationId !== null) cancelAnimationFrame(this.animationId);

    // 释放所有 GPU 资源（防止内存泄漏）
    this.scene.traverse(obj => {
      if (obj instanceof THREE.Mesh) {
        obj.geometry.dispose();
        if (Array.isArray(obj.material)) {
          obj.material.forEach(m => m.dispose());
        } else {
          obj.material.dispose();
        }
      }
    });

    this.controls.dispose();
    this.renderer.dispose();
    this.renderer.domElement.remove();
  }
}
```

### 7.5 坐标系注意事项

DXF/DWG 使用大坐标值（常见范围 0~100000），Three.js 默认 float32 在超过 ~8000 时出现精度抖动。解决方案：

1. 加载时将所有坐标**减去 boundingBox 中心**，归一化到原点附近
2. 相机位置同步偏移
3. 实体渲染前坐标变换：`vertex -= center`

// packages/moraya/src/index.ts
import { invoke } from '@tauri-apps/api/core';
import { createCadRenderer } from '@morcad/core';
import type { RendererPlugin } from '$lib/services/plugin/renderer-registry';

const adapter = {
  async readFile(absolutePath: string): Promise<Uint8Array> {
    // Tauri IPC：读取文件二进制内容（绕过 WebView 沙箱）
    const bytes = await invoke<number[]>('read_file_binary', { path: absolutePath });

**改动 2：`code-block-view.ts` 支持文件路径型渲染器**

当前所有渲染器的 `source` 是内容本身（JSON/DSL）。DXF 渲染器的 `source` 是文件路径，需要在调用 `render()` 前读取文件。建议扩展 `RendererPlugin` 接口：

```typescript
interface RendererPlugin {
  // ... 现有字段

  /**
   * 可选标记：声明此渲染器的 source 为文件路径而非内容本身。
   * code-block-view.ts 在调用 render() 前会先读取文件并将内容写入 source。
   * 若为 true，render() 收到的 source 已是文件内容（或由 render() 自行读取路径）。
   */
  isFilePathRenderer?: boolean;
}
```

```typescript
    const adapter: FileReaderAdapter = {
      async readFile(absolutePath: string): Promise<Uint8Array> {
        const buffer = await self.app.vault.adapter.readBinary(absolutePath);
        return new Uint8Array(buffer);
      },
      resolvePath(relativePath: string, docPath: string | null): string {
        if (!docPath) return relativePath;
        const dir = docPath.replace(/\/[^/]+$/, '');
        return self.app.vault.adapter.path.join(dir, relativePath);
      }
    };

    const cadRenderer = createCadRenderer(adapter);

    // 注册 dxf 和 dwg 代码块处理器
    for (const lang of ['dxf', 'dwg']) {
      this.registerMarkdownCodeBlockProcessor(
        lang,
        async (source, el, ctx) => {
          const isDark = document.body.classList.contains('theme-dark');
          const instance = await cadRenderer(el, source, ctx.sourcePath, isDark);

          // 注册清理函数（切换文件时销毁渲染器）
          this.registerEvent(
            this.app.workspace.on('file-open', () => instance.dispose())
          );
        }
      );
    }
  }
}
```

## 14. 关键技术备忘

### 14.1 内存管理（必须严格执行）

Three.js 不自动回收 GPU 资源，每次销毁 NodeView（切换文件/关闭标签）必须调用：

```typescript
// 正确的销毁顺序
scene.traverse(obj => {
  if (obj.geometry) obj.geometry.dispose();
  if (obj.material) {
    [obj.material].flat().forEach(m => m.dispose());
  }
});
controls.dispose();
renderer.dispose();
renderer.domElement.remove();
cancelAnimationFrame(animationId);
```

漏掉任何一步都会导致 GPU 内存泄漏，在多次打开/关闭 DXF 代码块后出现性能下降。

### 14.2 DWG WASM 首次加载体验

`@mlightcad/libredwg-web` WASM 文件约 5MB，首次加载有明显延迟（网络环境差时可能 10+ 秒）。必须：

1. 显示清晰的加载提示（"正在加载 DWG 解析模块，约 5MB…"）
2. 模块实例缓存在模块级变量，同文档后续 DWG 代码块复用
3. 若加载失败，显示友好错误："DWG 解析模块加载失败，请检查网络连接。"

### 14.3 坐标精度

DXF 工程图常用坐标范围：建筑图 0~100000，机械图 0~10000。Three.js 内部使用 32 位浮点，超过约 8192 的坐标会出现亚像素抖动。

**必须在加载实体前归一化坐标：**

```typescript
const offsetX = boundingBox.centerX;
const offsetY = boundingBox.centerY;
// 所有顶点坐标减去中心偏移
vertex.x -= offsetX;
vertex.y -= offsetY;
// 相机位置同步归零
camera.position.set(0, 0, 100);
```

### 14.4 AutoCAD 颜色系统

DXF 图层颜色使用 ACI（AutoCAD Color Index），0-256 的整数索引，不是 RGB。需要内置完整的 ACI → RGB 映射表（256 个条目）。特殊值：

- `0`：ByBlock（继承图块颜色）
- `256`：ByLayer（继承图层颜色）
- `1-7`：标准色（红/黄/绿/青/蓝/品红/白）
